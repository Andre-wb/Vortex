VORTEX / NODES

Registered nodes

Every peer the controller knows about. Clients read this list, verify the signature, and pick a node to connect to based on weight. A fresh sealed node gets weight 1.0; stale or unsealed nodes decay toward 0 so traffic flows to live, trusted peers only.

✓✓ DUAL Dual-verified

Registered with the controller AND sealed on-chain via Solana. Two independent witnesses confirm the code hash.

✓ CTRL Controller only

Registered with the HTTP controller. Signature proves the node owns its private key, but code integrity is not anchored on-chain.

⚠ UNVERIF Unverified bootstrap

Known from a config file or gossip. Not yet authenticated. Treat with caution — anyone can advertise a pubkey until proof arrives.
