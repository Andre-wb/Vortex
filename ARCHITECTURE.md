## Компоненты сервера

### `main.py` — точка входа
- Инициализация приложения FastAPI
- Подключение middleware: WAF, CSRF, Security Headers, Logging, TokenRefresh
- Регистрация роутеров
- Запуск через `uvicorn`

### `config.py` — конфигурация
- Чтение `.env`
- Автогенерация секретов JWT и CSRF при первом запуске
- Параметры: порты, таймауты, лимиты, пути к файлам

### `authentication/` — аутентификация
| Файл | Назначение |
|---|---|
| `auth.py` | Эндпоинты регистрации/входа: пароль + X25519 challenge-response |
| `auth_jwt.py` | Создание и проверка JWT, refresh-токены |

Rust-функции: `hash_password` (Argon2id), `verify_password`.

### `chats/` — чат и комнаты
| Файл | Назначение |
|---|---|
| `rooms.py` | Создание комнат, вступление, участники, ECIES-распределение ключей |
| `chat.py` | WebSocket E2E relay, WebRTC-сигнализация, загрузка файлов |

Взаимодействует с `peer/connection_manager.py` для широковещательной рассылки.

### `peer/` — пиры и федерация
| Файл | Назначение |
|---|---|
| `peer_registry.py` | UDP discovery (Rust / Python fallback), хранение пиров, federated-join, мультихоп |
| `connection_manager.py` | Управление WebSocket-соединениями, дедупликация, rate limiting |
| `federation.py` | Виртуальные комнаты, relay-менеджер, guest-login |

### `transport/` — транспортный стек
| Файл | Назначение |
|---|---|
| `transport_manager.py` | Выбор транспорта: Direct TCP → UDP punch → Wi-Fi Direct → BLE → Relay |
| `nat_traversal.py` | STUN-клиент, UDP hole punching |
| `ble_transport.py` | BLE: обнаружение, фрагментация пакетов |
| `wifi_direct.py` | Wi-Fi Direct для Linux / Windows |
| `routes.py` | API-эндпоинты управления транспортами |

### `security/` — безопасность
| Файл | Назначение |
|---|---|
| `crypto.py` | Унифицированный интерфейс к Rust / Python крипто |
| `waf.py` | Web Application Firewall: SQLi, XSS, rate limiting, блокировка IP |
| `middleware.py` | SecurityHeaders, CSRF, TokenRefresh |
| `secure_upload.py` | Валидация файлов, детектор аномалий, квоты |
| `key_exchange.py` | ECIES-шифрование / дешифрование для P2P |

### `files/` — файлы
| Файл | Назначение |
|---|---|
| `resumable.py` | Возобновляемая загрузка: чанки, SHA-256, сессии |

## База данных (SQLite WAL)

**Модели:** `User` · `Room` · `RoomMember` · `EncryptedRoomKey` · `PendingKeyRequest` · `Message` · `FileTransfer`

**PRAGMA:**
```sql
PRAGMA journal_mode = WAL;
PRAGMA foreign_keys = ON;
PRAGMA synchronous  = NORMAL;
```

WAL-режим обеспечивает конкурентный доступ: множество читателей не блокируют запись.

## Транспортный менеджер

```
Приоритет   Транспорт           Протокол                 Условие
──────────────────────────────────────────────────────────────────────
    4       Direct TCP/WS       HTTP / WebSocket          Оба узла в LAN
    3       UDP Hole Punch      UDP (STUN + punch)        Разные NAT
    2       Wi-Fi Direct        P2P (wpa_supplicant/WinRT) Нет AP
    1       BLE                 GATT (фрагментированный)  Wi-Fi недоступен
    0       Federation Relay    WebSocket через relay     Всё остальное
```

Менеджер проверяет транспорты сверху вниз и использует первый доступный.

## Мультихоп и федерация

```
Клиент A        Узел A (home)      Узел B (relay)     Узел C (target)
    │                 │                   │                   │
    │ POST            │                   │                   │
    │ /multihop-join  │                   │                   │
    │────────────────▶│                   │                   │
    │                 │ POST /guest-login │                   │
    │                 │──────────────────▶│                   │
    │                 │                   │ POST /join        │
    │                 │                   │──────────────────▶│
    │                 │◀──────────────────│◀──── токен ───────│
    │◀──── комната ───│                   │                   │
    │                 │                   │                   │
    │  WS сообщения   │   WS relay        │   WS relay        │
    │════════════════▶│══════════════════▶│══════════════════▶│
```

**Защита от петель:** каждый пакет несёт `msg_id` (UUID4) + `ttl=7`. Кеш `seen_ids` на 1000 записей (LRU) на каждом узле — дубликаты отбрасываются, TTL декрементируется.

## Безопасность

| Механизм | Реализация |
|---|---|
| **E2E шифрование** | X25519 + HKDF-SHA256 + AES-256-GCM — ключи комнат никогда не покидают клиент в открытом виде |
| **Аутентификация** | JWT HS256, refresh-токены с SHA-256 хэшем в БД |
| **CSRF** | Double Submit Cookie |
| **WAF** | Сигнатуры SQLi, XSS, path traversal, null bytes, command injection |
| **Rate limiting** | 120 запросов / мин на IP, автоблокировка на 1 час |
| **Файлы** | Проверка MIME, двойных расширений, zip-бомб, квоты |
| **Сетевые ограничения** | Федерация только с RFC 1918 IP |
