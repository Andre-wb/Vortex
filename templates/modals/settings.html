Связать устройство
Передайте ключи на новое устройство через 6-значный код. Ключи шифруются ECIES — сервер не может их прочитать.
Разделение ключа (M-of-N)
Разделите мастер-ключ на N частей по схеме Шамира. Любые M частей восстановят ключ. Каждая часть шифруется для конкретного доверенного контакта (ECIES). Сервер видит только зашифрованные блобы.

Журнал прозрачности ключей
Append-only лог всех изменений публичных ключей с цепочкой хешей (Merkle). Сервер подписывает каждую запись — подделка обнаруживается при аудите.

Как это работает: Ключи шифруются на вашем устройстве с помощью PBKDF2-SHA256 (600 000 итераций) + AES-256-GCM. Синхронизация между устройствами использует HKDF-SHA256 + AES-256-GCM. Кросс-подпись — HMAC-SHA256 от мастер-ключа. Каждое устройство имеет уникальный 6-emoji фингерпринт. Разделение ключа (SSSS) — схема Шамира над GF(256), M-of-N восстановление, каждая часть зашифрована ECIES. Федеративный бэкап распределяет зашифрованные шарды по узлам федерации — даже при потере основного сервера ключи восстановимы. Журнал прозрачности ключей — append-only Merkle-цепочка с HMAC-SHA256 подписями сервера, аудит обнаруживает подделку. Сервер хранит только зашифрованные данные и математически не может их расшифровать.
